Setelah anda menyelesaikan kode, tolong jawab pertanyaan dibawah ini ya =


1. Apa tujuan utama adanya interface dalam pemrograman berorientasi objek, dan bagaimana interface membantu menciptakan kontrak perilaku?
2. Mengapa interface dapat memiliki banyak implementasi, dan apa makna konseptual dari kemampuan tersebut dalam desain sistem?
3. Apa perbedaan mendasar antara extends dan implements dalam konteks interface, jika dilihat secara konsep tanpa melihat contoh kode?
4. Apa alasan diperkenalkannya default method dalam interface, dan bagaimana fitur ini memengaruhi keberlangsungan (compatibility) suatu sistem yang sudah berjalan?
5. Bagaimana konsep multiple interface memperluas fleksibilitas desain tanpa menimbulkan masalah kompleksitas yang biasa muncul pada multiple inheritance?

Jawaban :
1.Tujuan utamanya:
   Menentukan perilaku yang wajib dimiliki oleh class apa pun yang mengimplementasikannya.
  Interface = kontrak.
  Jika class implement interface itu artinya ia berjanji menyediakan method yang sama → membuat sistem lebih konsisten.
2.Karena interface hanya menentukan “apa yang harus dilakukan”, bukan “bagaimana caranya”.
  Makna konsepnya:
 - Banyak class bisa punya perilaku yang sama tapi cara kerja berbeda.
  - Ini mendukung polimorfisme dan desain fleksibel.
3.extends = Digunakan untuk mewarisi class atau interface. Turunannya mewarisi atribut + method induk.
  implements = Dipakai oleh class untuk memenuhi kontrak method dari interface. Tidak mewarisi atribut.
  Jadi konsepnya:
  extends = pewarisan (inheritance)
  implements = kontrak perilaku
4.Default method diperkenalkan agar:
  - Interface bisa ditambah method baru tanpa merusak class lama yang sudah mengimplementasikannya.
  - Memberikan implementasi dasar yang boleh di-override.
  Jadi sistem tetap backward compatible (kelas lama tidak error).
5.Multiple inheritance (dari class) sering menyebabkan konflik pewarisan.
  Java menghindarinya → diganti dengan multiple interface.
  Keuntungan:
  - Sebuah class bisa punya banyak “perilaku”
  - Tidak terjadi konflik atribut
  - Aman dari diamond problem
  - Desain lebih modular
